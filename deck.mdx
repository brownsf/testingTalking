#### Types of Testing

<ul>
  <Appear>

    <li>Unit</li>
    <li>Integration</li>
    <li>End to End</li>
  </Appear>

<Notes>
  Unit testing is a software testing where individual units (components) of a
  software are tested. The purpose of unit testing is to validate that each unit
  of the software performs as designed. A unit is the smallest testable part of
  any software. Integration tests are used to test multiple, bigger units
  (components) in interaction, and can sometimes even span multiple systems. The
  purpose of integration tests is to find bugs in the connections and
  dependencies between various components, such as: Passing invalid or
  incorrectly ordered arguments. e2e Testing how scenarios function on the
  product itself, by controlling the browser or the website. These tests usually
  ignore the internal structure of the application entirety and looks at them
  from the outside like on a black box.
</Notes>
